## Structure
- **YAML Frontmatter:** Must include `type: elicitation_manifest`.
- **1. Premise:** The starting point or user request that triggered the discovery.
- **2. Contextual Inquiries:** A list of questions asked to narrow the scope and their answers.
- **3. Problem Statement:** A concise definition of the problem being solved.
- **4. Success Criteria:** Quantitative and qualitative definitions of success.
- **5. Bounded Solution Space:** Explicit definition of what is in-scope vs. out-of-scope.
- **6. Requirements Mapping:** Links to future [Test Planning] acceptance criteria.
