## Structure
- **Requirement:** Every success criterion must map 1:1 to a specific test case in the [Test Plan](/docs/governance/protocol/test-planning.md).
- **Rule:** If a criterion cannot be tested, it must be refined or removed.
- **Verify:** Use the Knowledge Graph to link elicitation manifests directly to test planning.
