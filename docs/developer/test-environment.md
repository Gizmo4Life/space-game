# Test Environment & Linter Setup

This document describes the environment configuration required for local development and linter synchronization.

## 1. Linter Synchronization (clangd)

The project uses `.clangd` to configure the linter for C++20 and include paths.

### `.clangd` Configuration
Located in the project root, this file ensures the IDE correctly identifies headers from `src/` and external dependencies (SFML, EnTT, Catch2).

```yaml
CompileFlags:
  Add: [-std=c++20, -I/Users/Dan/repos/space-game/src, -I/Users/Dan/repos/space-game/build/_deps/catch2-src/src, -I/Users/Dan/repos/space-game/build/_deps/catch2-build/generated-includes]
```

## 2. Compilation Database

The linter relies on `compile_commands.json` generated by CMake.

### Setup Instructions
1. **Generate**: Enable `CMAKE_EXPORT_COMPILE_COMMANDS` during configuration.
   ```bash
   cmake -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
   ```
2. **Link**: Create a symlink in the project root so the IDE can find it.
   ```bash
   ln -s build/compile_commands.json .
   ```

## 3. Catch2 Requirements

To avoid template decomposition ambiguities in the linter:
- **Standard**: Always use double parentheses for `REQUIRE` statements.
  - `REQUIRE((expression));`
- **Avoid**: Implicit boolean casts or naked comparisons that might trigger complex template deduction.

## 4. Local Build Directory

Testing should be performed in the `/Users/Dan/repos/space-game/build` directory to ensure all local dependencies (like Catch2) are resolved without network dependency.
